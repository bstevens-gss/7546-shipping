1. Executive Summary

Hultec’s shipping team currently relies on the Shipping Dashboard (project 7546) for load planning and picker assignment via GS Mobile. When a sales order line needs to be partially fulfilled on different dates, there is no way to split that line from within the dashboard — users must exit to the classic Order Entry screen, manually create a new line, adjust quantities, and return to the dashboard. This disrupts workflow and introduces data-entry errors.

This engagement adds two new capabilities directly to the Shipping Dashboard:

  • Split SO Line — A right-click context menu item on the Open Orders and Due Today/Past Due grids that opens a modal form. Users enter a split quantity and new due date; the system creates a new SO line via ORDADCMD, updates the original line via the Sales.UpdateOrderLinesByFile CallWrapper, and enforces business rules against ship schedules and staged shipments.
  • Edit Sales Order — A right-click context menu item that launches the classic Order Entry edit screen (Sales.EditSalesOrderInClassicScreen) for the selected order, with permission and lock checks, without leaving the dashboard.

Additionally, a shared dictionary optimization eliminates 6 duplicate SQL queries per load/refresh cycle, improving dashboard performance.

Process Change — Replaces Existing Workflows

By adopting the Split SO Line approach on the Shipping Dashboard, Hultec will retire two existing processes:

  • Custom Reservation Program — No longer needed; order line splitting within the dashboard replaces the reservation workflow for allocating partial quantities to different ship dates.
  • Blanket Sales Order and Release — No longer needed; splitting lines with specific quantities and due dates directly on standard sales orders replaces the blanket order/release process.

This consolidation simplifies the shipping workflow into a single tool, reducing training requirements and eliminating the need to maintain separate programs.

2. Scope of Work

2.1 — Split SO Line Feature

Context Menu Integration

A new “Split SO Line” context menu item is added to both the Open Orders (GsGCAllShip) and Due Today & Past Due (GsGCDue) grids. The item appears when right-clicking on any row, regardless of mobile picklist settings.

Modal Split Form

Clicking the menu item opens a modal form (frmSplitLine) displaying all checked rows in a grid (GsGCSplit). The grid shows read-only context columns (customer, order number, line number, part, description, ship-to details, open quantity) and two editable yellow-highlighted columns:

  • Qty to Split — Numeric field; must be greater than 0 and strictly less than the open quantity
  • New Due Date — Date picker for the new split line’s promise date

Save Processing

On save, each selected line is processed with the following validation and creation steps:

  1. Order Lock Check — Verifies the order is not locked by another user
  2. Ship Schedule Validation — Blocks the split if any record exists in V_OE_SHIP_SCHED for the order line
  3. Staged Shipment Validation — Blocks the split if any record exists in STAGING_QUANTITY for the order line
  4. New Line Creation — Determines next available line number, builds a fixed-width upload file, and calls ORDADCMD via Launchgsssync
  5. User Field Traceability — Dynamically looks up the OE_USER_FLDS_LN table for a field labeled “SPLIT FROM LINE” and writes the original line number to that user field on the new line (skips gracefully if not configured)
  6. Date Copy — Copies DATE_ORDER from the original line to the new line
  7. Line Updates via CallWrapper — Builds a two-line CSV file processed by a single Sales.UpdateOrderLinesByFile CallWrapper invocation:
    • Line 1: Updates the original line’s quantity, price (6-decimal precision), and recalculated weight
    • Line 2: Sets DATE_ITEM_PROM (CSV position 16) on the new split line

Grid Serialization

The split form grid supports user layout persistence (column widths, order, sort, filter) via GSS Registry key GVSPLIT. A “Reset Grid Serialization” context menu item is available to restore default layout.

2.2 — Edit Sales Order Feature

A new “Edit Sales Order” context menu item is added below “Split SO Line” on the same grids. When clicked:

  1. Checks IPM permission (Order Entry > Edit > Sales Orders, level 1)
  2. Verifies the order is not locked
  3. Disables the parent tab to prevent concurrent actions
  4. Launches Sales.EditSalesOrderInClassicScreen CallWrapper for the selected order
  5. Re-enables the parent tab on return

2.3 — Performance Optimization

Six dictionary SQL queries that were duplicated between LoadOpenOrders and LoadLoad are consolidated into shared subroutines (LoadSharedDictionaries / CloseSharedDictionaries). Dictionaries are loaded once per refresh cycle and reused across both data loads, eliminating 6 redundant SQL queries per cycle.

Business Rules

The following rules are enforced and cannot be overridden by the user:

  • Split quantity must be strictly less than open quantity (no full-quantity splits)
  • Lines with any ship schedule record (open or closed) cannot be split
  • Lines with any staged shipment record cannot be split
  • Locked orders are skipped (per-line, allowing partial batch processing)
  • There is no undo — corrections must be made via the Edit Sales Order screen
