Title
Order Entry & Shipping Enhancements — Split SO Line, Edit Sales Order, Performance Optimization
Description

Hultec’s shipping team uses the Shipping Dashboard (project 7546) for load planning and picker assignment. This enhancement adds the ability to split sales order lines directly from within the dashboard, eliminating the need to exit to the classic Order Entry screen. A new “Split SO Line” context menu item opens a modal form where users enter a split quantity and new due date. The system creates a new SO line via ORDADCMD, updates the original line via the Sales.UpdateOrderLinesByFile CallWrapper, and enforces business rules against ship schedules and staged shipments.

Additionally, an “Edit Sales Order” context menu item launches the classic Order Entry edit screen without leaving the dashboard, and a shared dictionary optimization eliminates 6 duplicate SQL queries per load/refresh cycle.

Process Change — Replaces Existing Workflows

By adopting the Split SO Line approach, Hultec will retire two existing processes:

  • Custom Reservation Program — No longer needed; order line splitting within the dashboard replaces the reservation workflow.
  • Blanket Sales Order and Release — No longer needed; splitting lines with specific quantities and due dates on standard sales orders replaces the blanket order/release process.

This consolidation simplifies the shipping workflow into a single tool, reducing training requirements and eliminating separate programs.

Specifications

Scope of Work

  1. Split SO Line context menu — Add “Split SO Line” to right-click menus on Open Orders and Due Today/Past Due grids
  2. Split SO Line modal form — Grid displaying selected lines with editable Qty to Split and New Due Date columns (yellow-highlighted)
  3. Split save processing — Order lock check, ship schedule validation, staged shipment validation, new line creation via ORDADCMD, original line update + promise date via Sales.UpdateOrderLinesByFile CallWrapper (two-line CSV), DATE_ORDER copy, weight recalculation
  4. Dynamic user field traceability — Lookup “SPLIT FROM LINE” label in OE_USER_FLDS_LN, write original line number to mapped user field on new line
  5. Grid serialization — Save/restore split form grid layout via GSS Registry (GVSPLIT key)
  6. Edit Sales Order context menu — Right-click item launching Sales.EditSalesOrderInClassicScreen with permission and lock checks
  7. Shared dictionary optimization — Consolidate 6 duplicate SQL queries into LoadSharedDictionaries/CloseSharedDictionaries subroutines

Validation Logic

Rule Check Behavior
Split qty bounds QTY_SPLIT > 0 AND QTY_SPLIT < QTY_BO Block entire save with message
Order lock OrderLocked subroutine Skip line, continue batch
Ship schedule COUNT(*) FROM V_OE_SHIP_SCHED Skip line, continue batch
Staged shipment COUNT(*) FROM STAGING_QUANTITY Skip line, continue batch
IPM permission CheckUserAccessIPM(132,1) Block action with message
