Chapter 1

The Problem

When the dashboard becomes a detour

Hultec's shipping team lives in the Shipping Dashboard — a custom GAB application built on Global Shop Solutions that handles load planning, picker assignment via GS Mobile, and daily shipping operations across five tabs of real-time data.

But when a sales order line needs to be partially fulfilled on different dates — a common scenario in Hultec's manufacturing workflow — the dashboard hits a wall. There is no way to split an order line from within the tool where all the shipping context lives.

The Pain Point

To split a single order line, users must:

  • Exit the Shipping Dashboard entirely
  • Navigate to the classic Order Entry screen
  • Manually create a new line with the split quantity
  • Adjust quantities on the original line
  • Return to the Shipping Dashboard and refresh

This context-switching disrupts workflow, introduces data-entry errors, and costs significant time across the team's daily operations.

Unified Workflow

  3. Enter quantity and date
  4. Click Save

4 steps • 1 screen • Validated & safe

On top of the split workflow problem, Hultec also maintained two separate legacy processes that existed solely because this capability wasn't available in the dashboard:

Custom Reservation Program

Used to allocate partial quantities to different ship dates — replaced by direct line splitting.

Blanket Sales Order and Release

Used to manage orders with multiple ship dates — replaced by splitting lines with specific quantities and dates on standard sales orders.

Chapter 2

The Solution

Three deliverables, one unified shipping experience

Rather than building a separate tool, we brought the capabilities directly into the Shipping Dashboard that the team already uses every day. Three targeted deliverables address the core problem and improve overall performance.

Split SO Line

Right-click any order line in the Open Orders or Due Today grids to open a modal split form. Enter the quantity to split and a new due date. The system creates a new SO line via ORDADCMD, updates the original via CallWrapper, and enforces full business rule validation.

GAB Feature • Modal Form • Batch Processing

Edit Sales Order

A new context menu item launches the classic Order Entry edit screen directly from the dashboard. Permission and lock checks prevent unauthorized or conflicting edits. The dashboard tab is disabled during editing to prevent concurrent actions.

GAB Feature • CallWrapper Integration

Performance Optimization

Six dictionary SQL queries duplicated between LoadOpenOrders and LoadLoad are consolidated into shared subroutines. Dictionaries load once per refresh cycle, eliminating 6 redundant database round-trips.

Chapter 3

How It Works

Architecture, integration, and safety nets

System Architecture

The Shipping Dashboard is a GAB script (.g2u) that reads data via ODBC and writes through GSS CallWrappers — the supported API layer for transactional changes in Global Shop Solutions.

Architecture Overview — The Shipping Dashboard reads via ODBC and writes through GSS CallWrappers

Split SO Line — Processing Flow

When a user saves a split, each selected line passes through a rigorous validation pipeline before any data is written. Lines that fail validation are skipped; successful lines continue processing. This allows batch splits across multiple orders.

Split SO Line Processing Flow — Each line validated independently, enabling partial batch success

Business Rule Enforcement

Every split operation passes through four mandatory validation checks. These rules cannot be overridden by the user and exist to protect data integrity.

Split Quantity Bounds
Qty must be greater than 0 and strictly less than the open quantity. No full-quantity or zero splits allowed.
Blocks entire save
Order Lock Check
Verifies the order is not locked by another user. Identifies the locking user by name.
Skips line, continues batch
Ship Schedule Validation
Blocks if any record exists in V_OE_SHIP_SCHED for the order line (open or closed).
Skips line, continues batch
Staged Shipment Validation
Blocks if any record exists in STAGING_QUANTITY for the order line.
Skips line, continues batch

CallWrapper Integration

The split operation uses a two-line CSV file processed by a single Sales.UpdateOrderLinesByFile CallWrapper invocation. Line 1 updates the original order line (reduced quantity, price at 6-decimal precision, recalculated weight). Line 2 sets the promise date on the new split line.

2 Order Number {OrderNo} {OrderNo}
3 Order Line {OrigLineNo} {NewLineNo}

Chapter 4

The Interface

Designed to feel native — not bolted on

Every new feature integrates through the existing right-click context menu pattern that users already know. No new buttons, no new navigation — just two new items at the bottom of the familiar menu.

Context Menu — New items appear below the separator on Open Orders and Due Today/Past Due grids

Split SO Line Modal Form

The split form displays all checked rows in a grid with read-only context columns and two editable yellow-highlighted columns. Users can split multiple lines across different orders in a single batch operation.

Split SO Line Modal — Editable Qty to Split and New Due Date columns (yellow) with read-only context
Traceability Built In

When the user field "SPLIT FROM LINE" is configured in OE_USER_FLDS_LN, the system automatically writes the original line number to that field on the new split line. This provides an audit trail without requiring any manual data entry. If the field isn't configured, the feature degrades gracefully — the split still works, just without the traceability link.

Grid Serialization

The split form grid supports user layout persistence (column widths, order, sort, filter) via GSS Registry key GVSPLIT. Users can customize the grid to their workflow, and a "Reset Grid Serialization" context menu item restores the default layout.

Chapter 6

The Impact

Consolidation, speed, and confidence

This project consolidates Hultec's shipping workflow into a single tool. The team no longer maintains separate reservation programs or blanket order processes. Everything happens in the dashboard they already use.

Process Consolidation

Two legacy processes — the Custom Reservation Program and the Blanket Sales Order and Release workflow — are retired. This reduces training requirements, eliminates the need to maintain separate programs, and simplifies Hultec's operational footprint.

Performance Improvement

The shared dictionary optimization eliminates 6 redundant SQL queries per dashboard refresh cycle. For a tool used continuously throughout the workday, this translates to measurable responsiveness improvement.

Safety by Design

Unlike the manual process, every split is validated against ship schedules, staged shipments, and order locks before any data is written. Batch operations process lines independently, so one invalid line doesn't block the rest. There is no undo — but corrections are one right-click away via the Edit Sales Order feature.
