Title
Shipping Enhancements — Pick Lists, Packing Lists, Label Printing & Load Manifest
Description

Wisconsin Steel & Tube’s (WST) shipping team requires a series of enhancements to the Shipping Dashboard (project 7546) covering pick list printing, packing list generation, customer-specific label printing (including multiple copies per container), and load manifest reporting. These four items address workflow gaps in the current shipping process — from assigning orders to pre-pick loads and printing pick lists by individual order line, to generating service pack lists for outside purchase orders, enabling customer-driven label configurations (building on the extended 9001 shipping label pre-proc already delivered, ARC 6907), and correcting weight calculations on load manifests.

All enhancements are scoped as modifications to the existing 7546 Shipping Dashboard and its associated Crystal Reports. The work will be performed on a Time & Materials basis at $225.00/hr, estimated at 27 hours.

Assumptions

  • The existing Shipping Dashboard (project 7546, GAB_7546_OE_ShippingReview_Load.g2u) has been tested, accepted, and is functioning as designed with its current feature set. This engagement covers only the modifications described in the Scope of Work. Pre-existing defects, untested functionality, or issues unrelated to the modifications defined herein are excluded from this scope and will be addressed as a separate engagement if discovered.
  • WST’s GSS environment is version 2020.1 or higher with Crystal Reports integration functional
  • Existing pick list, packing list, and load manifest Crystal Reports are accessible and modifiable
  • Label printing infrastructure (printer assignments, label templates) is operational in WST’s environment
  • The extended pre-proc mechanism for 9001 labels is available in the current GSS version
  • WST will provide sample data and test scenarios for customer-specific label configurations
  • Access to WST’s database (ODBC) for testing staged order scenarios and lot/bin data

Prerequisites

  • Shipping Dashboard (7546) deployed and accessible in WST’s environment
  • GSS version ≥ 2020.1 with Crystal Reports and label printing subsystem
  • ODBC connectivity to Actian Zen database
  • Printer assignments configured for pick list and label output
  • Sample customer-specific label requirements and naming conventions from WST
Specifications

Scope of Work

  1. Pick List Printing Enhancement (Report ID 95) — Add a context menu item to assign orders to a pre-pick load and print pick lists by individual order line. Each order line prints as a separate pick list, filtered to show only lines in the designated load plan (e.g., load 9999), with appropriate page breaks and printer assignment.
  2. Packing List with Certs Enhancement — Modify the packing list with certs process so that when printing for outside purchase orders, a service pack list also prints for each line on the load. Include the ability to manipulate the output data as needed (e.g., removing lot bin information from the service pack list).
  3. Customer-Specific Label Printing & Multiple Copies — Build on the extended 9001 shipping label pre-proc (ARC 6907) already delivered for this site. WST will design their own customer-specific 9001 shipping labels. Configure naming conventions for integration. Test and implement multiple identical labels per container (e.g., for all sides of a pallet) based on customer requirements, using a parameter in the label name to override the default number of copies.
  4. Load Manifest Weight Calculation — Modify the load manifest Crystal report to display the total weight per line by multiplying staged quantity by part weight from the inventory master. Ensure the manifest only shows validated lines.

Testing Plan

TC-01: Pick List Printing by Line

Objective:

  • Verify that the context menu item assigns selected order lines to a pre-pick load and prints a separate pick list per line, filtered to the designated load plan

Steps:

  • Open the Shipping Dashboard and navigate to a load with multiple order lines assigned to load plan 9999
  • Right-click and select the new “Assign to Pre-Pick Load & Print Pick Lists” context menu item
  • Confirm the print dialog and verify output at the assigned printer

Expected Results:

  • Each order line in the load plan prints as a separate pick list with its own page break
  • Only lines belonging to the specified load plan (e.g., 9999) appear on the pick lists
  • Output is routed to the correct printer assignment

TC-02: Packing List with Certs — Service Pack for Outside POs

Objective:

  • Verify that printing a packing list with certs for outside POs also generates a service pack list for each line on the load, with lot bin info removed as configured

Steps:

  • Stage an order that includes outside purchase order lines on a load
  • Initiate the packing list with certs print process for the load
  • Review the printed output for both the packing list and the accompanying service pack lists

Expected Results:

  • A service pack list prints for each outside PO line on the load
  • Lot bin information is excluded from the service pack list output
  • The standard packing list with certs prints normally for non-outside-PO lines

TC-03: Customer-Specific Label Printing & Multiple Copies

Objective:

  • Verify customer-specific 9001 label configuration (ARC 6907 extended pre-proc foundation) prints the correct template by lot, with lookup fallback to default
  • Verify the label name copy-count parameter overrides the default and produces the expected number of identical labels per container

Steps:

  • Configure a customer-specific label using the documented naming convention; stage an order for that customer with multiple lots
  • Print labels from shipping by lot and verify the correct template is selected
  • Add the copy-count parameter to the label name (e.g., 4 copies for pallet sides), stage containers/pallets, print, and count labels per container

Expected Results:

  • Customer-specific templates resolve per naming convention; customers without a specific template fall back to the default label
  • Per-container copy count matches the parameter; all copies are identical in content
  • Labels without the copy-count parameter print the standard single copy

TC-04: Load Manifest Weight Calculation

Objective:

  • Verify that the load manifest Crystal report displays total weight per line as staged quantity multiplied by part weight, and only shows validated lines

Steps:

  • Create a load with multiple lines, some validated and some not validated
  • Note the staged quantity and part weight (from inventory master) for each validated line
  • Print the load manifest and compare the displayed weights to the manual calculation

Expected Results:

  • Each line’s total weight equals staged quantity × part weight from inventory master
  • Only validated lines appear on the manifest
  • Non-validated lines are excluded from the report and from the total weight

Technical Scope of Work

1. Pick List Printing Enhancement (Report ID 95)

  • Add context menu item to the Shipping Dashboard grid (GAB_7546_OE_ShippingReview_Load.g2u)
  • Implement pre-pick load assignment logic, writing load plan identifier (e.g., 9999) to staging records
  • Invoke pick list Crystal Report with line-level filtering — pass load plan number as selection parameter
  • Configure report grouping to force page breaks per order line
  • Route output to the designated printer via printer assignment lookup

2. Packing List with Certs — Service Pack List

  • Modify the packing list with certs print process to detect outside PO lines on the load
  • For each outside PO line, trigger an additional service pack list report
  • Implement data manipulation layer to suppress lot bin information on the service pack output
  • Coordinate print sequencing so packing list and service pack list print in logical order per line

3. Customer-Specific Label Printing & Multiple Copies

  • The extended 9001 shipping label pre-proc (ARC 6907), already delivered for this site, is the foundation; extend configuration and shipping integration on that mechanism
  • WST will design their own customer-specific 9001 shipping labels; document and configure the naming conventions required for integration
  • Configure the extended pre-proc for 9001 labels to resolve the correct template at print time based on customer
  • Implement label lookup logic: customer-specific → default fallback
  • Deliver label template files and configuration instructions for WST’s shipping team
  • Implement a label name parameter convention to encode copy count (e.g., Label_x4 for 4 copies)
  • Parse the parameter at print time and override the default copy count in the print loop
  • Ensure backward compatibility — labels without the parameter print the standard single copy

4. Load Manifest Weight Calculation

  • Modify the load manifest Crystal Report to add a calculated field: Total_Weight = STAGING_QUANTITY.QTY_STAGED × INVENTORY_MSTR.WEIGHT
  • Add record selection formula to filter for validated lines only
  • Add report footer with grand total weight across all validated lines
